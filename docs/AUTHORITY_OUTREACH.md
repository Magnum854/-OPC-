# 深大 OPC 社区权威引用与外联素材

本文件用于争取真实、相关且可核验的校内外报道与反向链接。目标不是批量换链，而是让深圳大学相关单位、活动合作方和真实媒体在介绍社区或项目时，准确引用社区名称和官网 canonical。

## 统一引用口径

- 推荐名称：深大 OPC 社区（SZU OPC Community）
- 推荐官网：https://szu-opc.pages.dev/
- 推荐解释页：https://szu-opc.pages.dev/articles/what-is-opc
- 实践文章中心：https://szu-opc.pages.dev/articles/
- 项目档案：https://szu-opc.pages.dev/projects
- 深大首届 Vibe Coding 黑客松活动档案：https://szu-opc.pages.dev/events/szu-vibe-coding-hackathon-2026
- 一句话介绍：深大 OPC 社区面向深圳大学学生、教师与研究者、校友及合作伙伴，围绕 One Person Company、Vibe Coding、AI 应用和真实项目开展实践、共创与复盘。

不要使用“深圳大学官方 OPC 社区”“校级组织”“深圳首个”“最大”或“唯一”等未经公开材料证实的表述。本站不是深圳大学门户网站；学校授权、主办、指导或组织关系以可核验公告为准。

## 优先外联顺序

1. 深圳大学创新创业教育中心：争取发布社区项目、活动复盘或成员成果报道，并在正文中链接官网。
2. 与具体项目或活动直接相关的学院、实验室和教师主页：只提交可核验的事实、日期、参与者与成果。
3. 实际合作方、参赛或获奖平台：请其在合作案例或活动回顾中引用官网。
4. 校园媒体、真实行业媒体：优先提供有数据、有作品、有现场材料的新闻线索。
5. 已核实的公众号、视频号、Bilibili、小红书等官方社区账号：保持名称、头像、简介和官网链接一致。

外联前先确认对方与内容确有关系。不要购买链接、群发无关站点、建立关键词堆砌的目录页，或要求对方伪造主办与认证关系。

## 发布前证据包

每次外联至少准备以下信息，缺失时不要声称相应事实：

- 活动或项目准确名称、时间、地点；
- 主办、指导、承办与合作关系的公开证明；
- 可公开的参与者姓名与授权；
- 项目截图、演示地址或现场照片的使用授权；
- 可量化成果及其统计口径；
- 官网中与报道相匹配的 canonical 页面；
- 对方发布后的最终 URL、标题和发布日期。

## 获得链接后的维护

- 检查链接是否指向 HTTPS canonical，而不是旧域名、带 `.html` 的地址或 hash 片段。
- 检查报道中的名称和组织关系是否准确。
- 只有在确认账号确由社区控制后，才把它加入网站的 `sameAs`。
- 在 Search Console 的“链接”和“效果”报告中按月观察引用域、品牌词展示与点击，不以单日排名判断结果。
- 如果报道被删除、改址或出现错误，记录并联系原发布方修正；不要自行伪造镜像页面。
